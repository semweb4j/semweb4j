47      /**
48       * OWL Full does not put any constraints on annotations in an ontology. 
49       * OWL DL allows annotations on classes, properties, individuals and 
50       * ontology headers, but only under the following conditions:
51       *
52  	 * - The sets of object properties, datatype properties, annotation properties 
53  	 * and ontology properties must be mutually disjoint. Thus, in OWL DL 
54  	 * dc:creator cannot be at the same time a datatype property and an 
55  	 * annotation property.
56  	 * - Annotation properties must have an explicit typing triple of the form:
57  	 * -- AnnotationPropertyID rdf:type owl:AnnotationProperty .  
58  	 * - Annotation properties must not be used in property axioms. Thus, in 
59  	 * OWL DL one cannot define subproperties or domain/range constraints for 
60  	 * annotation properties.
61  	 * - The object of an annotation property must be either a data literal, a 
62  	 * URI reference, or an individual.
63       */
64      public static final URI AnnotationProperty = toURI("AnnotationProperty");
65      /**
66       * An owl:imports statement references another OWL ontology containing 
67       * definitions, whose meaning is considered to be part of the meaning of 
68       * the importing ontology. Each reference consists of a URI specifying from 
69       * where the ontology is to be imported. Syntactically, owl:imports is a 
70       * property with the class owl:Ontology as its domain and range.
71       *
72       * The owl:imports statements are transitive, that is, if ontology A 
73       * imports B, and B imports C, then A imports both B and C.
74       */
75      public static final URI imports = toURI("imports");

195      * semantics other than that given by the RDF(S) model theory. However, it

204     /**
205      * An owl:backwardCompatibleWith statement contains a reference to another 
206      * ontology. This identifies the specified ontology as a prior version of 
207      * the containing ontology, and further indicates that it is backward 
208      * compatible with it. In particular, this indicates that all identifiers 
209      * from the previous version have the same intended interpretations in the 
210      * new version. Thus, it is a hint to document authors that they can safely 
211      * change their documents to commit to the new version (by simply updating 
212      * namespace declarations and owl:imports statements to refer to the URL of 
213      * the new version). If owl:backwardCompatibleWith is not declared for two 
214      * versions, then compatibility should not be assumed.
215      *
216      * owl:backwardCompatibleWith has no meaning in the model theoretic 
217      * semantics other than that given by the RDF(S) model theory.
218      * 
219      * owl:backwardCompatibleWith is a built-in OWL property with the class 
220      * owl:Ontology as its domain and range.
221      *
222      * NOTE: owl:backwardCompatibleWith is an instance of owl:OntologyProperty. 
223      */
224     public static final URI backwardCompatibleWith = toURI("backwardCompatibleWith");
225     /**
226      * An owl:incompatibleWith statement contains a reference to another 
227      * ontology. This indicates that the containing ontology is a later version 
228      * of the referenced ontology, but is not backward compatible with it. 
229      * Essentially, this is for use by ontology authors who want to be explicit 
230      * that documents cannot upgrade to use the new version without checking 
231      * whether changes are required.
232      *
233      * owl:incompatibleWith has no meaning in the model theoretic semantics 
234      * other than that given by the RDF(S) model theory.
235      *
236      * owl:incompatibleWith is a built-in OWL property with the class 
237      * owl:Ontology as its domain and range.
238      *
239      * NOTE: owl:backwardCompatibleWith is an instance of owl:OntologyProperty. 
240      */
241     public static final URI incompatibleWith = toURI("incompatibleWith");
